98 /** \brief base class for 2D plotter classes (used by the plotter widget JKQTPlotter)
99  * \ingroup jkqtpplotterclasses
100  *
101  * This class implements basic functionalities for the plotter classes. Those are:
102  * -# data storage using a JKQTPDatastore object
103  * -# coordinate transforms
104  * -# a set of properties for the graphs (colors, widthes ...) and also a system (see getNextStyle() to automatically
105  * choose a drawing style for different graphs.
106  * -# plot a set of overlay elements (may be used for fast plotting of indicators onto a complex plot)
107  * -# drawing the coordinate axes, grids ... (logarithmic and linear)
108  * -# saveing and printing the resulting plots
109  * .
110  *
111  * This class is NOT a widget, if you need a plotting widget, use JKQTPlotter. This class may be used to
112  * plot using a JKQTPEnhancedPainter.
113  *
114  * \see JKQTPlotter
115  *
116  * \section jkqtplotter_base_datastore Data Storage
117  * As already mentioned this class does not provide means to draw graphs, but it contains a basic mechanism to associate
118  * data with it. This is imeplemented, using a JKQTPDatastore object together with some convenience access methods. No further
119  * data access is implemented and child classes are NOT forced to use this datastore. Hide it if you want. JKQTPlotter shows
120  * how to use it. This class implement a graph management, where graphs simply point to a set of columns inside the datastore
121  * that contain the actual plot data!
122  *
123  * If you call the \link JKQTBasePlotter::JKQTBasePlotter() constructor \endlink with no arguments, it will create an internal
124  * datastore object and you can start adding data by using getDatastore(). If you have an external JKQTPDatastore object you can
125  * give it as parameter to the constructor or use one of the other methods:
126  * - useExternalDatastore(): \copybrief JKQTBasePlotter::useExternalDatastore()
127  * - useAsInternalDatastore(): \copybrief JKQTBasePlotter::useAsInternalDatastore()
128  * - useInternalDatastore(): \copybrief JKQTBasePlotter::useInternalDatastore()
129  * - forceInternalDatastore(): \copybrief JKQTBasePlotter::forceInternalDatastore()
130  *.
131  *
132  *
133  *
134  * \section jkqtplotter_base_systems_baseplotter Coordinate Systems and Transformations
135  * These topics are discussed in the help for JKQTPCoordinateAxis. There is a set of coordinate transform
136  * methods (x2p(), y2p(), p2x(), p2y() which only call the respective methods in xAxis and yAxis objects.
137  *
138  *
139  * \section jkqtplotter_base_grids_baseplotter Axis Ticks and Grids
140  * - The plotting of coordinate axes and grids, as well as coordinate transforms is done by
141  * JKQTPCoordinateAxis descendents (see documentation there)
142  * .
143  * If you want to set the axis properties, use getXAxis() or getYAxis() to get a pointer to the axis objects which then
144  * may be used to set the axis properties.
145  *
146  *
147  * \section jkqtbaseplotter_appearance_and_style Appearance & Styling of the Graph
148  * \subsection jkqtplotter_base_key Graph Keys
149  * This class provides means to plot a key together with the functions. The plotting is only partially implemented in this base class, and has to be
150  * implemented by child classes, as no graph management, that could provide a title for a specific graph, is implemented here. Key Plotting is
151  * performed in the plotKey() method. This method basically draws a frame and background and then calls plotKeyContent() to draw the actual contents.
152  * The size of the key is determined by a call to getKeyExtent(), while it's position and plotting style is determined by the \c key_... parameters.
153  * If getKeyExtent() returns 0 for height or width, no key is plotted. The \c key_... parameters are designed to be rather common for any usual kind
154  * of key, so any implementation of key should be able to use them to parametrize the output. Note however that additional parameters might be defined
155  * in child classes.
156  *
157  * \image html plot_key.png
158  *
159  * As it may be convenient to position the key completely outside the graph this class may change the plotBorders to make the key fit outside
160  * SO if you use a keyPosition \b outside the graph you should NOT keep any space for the key in the margins, as the program will change them to
161  * accomodate the size of the key. If you select an \b inside key position the key will be plotted OVER the graph, i.e. the margins won't be changed
162  * Note that the margin change is internal and not visible in the class interface!
163  *
164  * There is also a possibility to determine the size of the key automatically, so all text fits in. This is activted by the property keyAutosize ( \copybrief keyAutosize ). If this
165  * is \c true the function getKeyExtent() has to check the width of every key item and take it into account when calculating the width and height of the
166  * key content. By default this feature is switched ON.
167  *
168  *
169  * \subsection jkqtplotter_base_aspectratios Aspect Ratios
170  * First note that this functionality is only available and activated if both axes are linear!
171  *
172  * You can set two different aspect ratios:
173  * - The ratio of plotWidth/plotHeight (setAspectRatio(), setMaintainAspectRatio()) will keep the plots pixel-width and height at a certain value.
174  * - The ratio of (xmax-xmin)/(ymax-ymin) (setAxisAspectRatio(), setMaintainAxisAspectRatio()) will keep the displayed axis ranges in a certain ratio.
175  * .
176  * So to achieve different effects, use these combinations:
177  * - you have a 200x100 range where each 1x1-pixel should have an aspect ratio of 4:
178  * \code
179  * setAspectRatio(200.0/100.0);
180  * setAxisAspectRatio(4.0*getAspectRatio());
181  * \endcode
182  * .
183  *
184  *
185  * \section jkqtbaseplotter_dataexport_print Printing, Saving & Exporting
186  *
187  * \subsection jkqtplotter_base_saveprint Saving and Printing Graphs
188  * This class implement a set of function to save and print the graphs:
189  * - print() prints the graph on a QPrinter object
190  * - saveAsPixelImage() saves the plot into a pixel image file (PNG, TIFF, ... formats, as supported by Qt)
191  * - saveAsPDF() saves the graph as a PDF file (using the Qt printing engine)
192  * - saveAsPS() saves the graph as a PDF file (using the Qt printing engine)
193  * - saveAsSVG() saves the graph as a SVG file (using the Qt SVG library)
194  * - saveImage() saves the graph
195  * .
196  * You can also copy the contents of the plot into the clipboard:
197  * - copyPixelImage()
198  * .
199  *
200  * \subsection jkqtplotter_base_dataexport Exporting Graph Data
201  * This class implement a set of function to save the data of the graphs:
202  * - saveData() saves the data of the plot
203  * - saveAsCSV() saves the data of the plot as comma separated values
204  * - saveAsSYLK() saves the data of the plot as SYLK spreadsheet
205  * - saveAsDIF() saves the data of the plot as data interchange format file
206  * - saveAsMatlab() saves the data of the plot as a CSV file suitable for Matlab
207  * - saveAsSemicolonSV() saves the data of the plot as a Semicolon Separated Values (SSV)
208  * - saveAsTabSV() saves the data of the plot as a Tabulator Separated Values (CSV) file
209  * - saveAsGerExcelCSV() saves the data of the plot as a Text file (SSV) suitable for german excel, i.e. with comma as decimal separator
210  * .
211  * You can also copy the graphs' data into the clipboard:
212  * - copyData()
213  * - copyDataMatlab()
214  * .
215  *
216  *
217  * \subsection JKQTBASEPLOTTER_GRIDPRINTING Grid-Printing / Layouting Several Graphs
218  * As one often want's to combine different graphs, there is a possibility to combine this graph with more other graphs.
219  * To do so one can think of the graphs to be part of a grid where each graph is contained in one cell. By default this
220  * mechanism is deactivated. You can activate it by calling setGridPrinting(true). Then you can set the position of the
221  * current graph by calling setGridPrintingCurrentX() and setGridPrintingCurrentY(). Add additional graphs by calling
222  * addGridPrintingPlotter(). The position of the current graph is 0,0 by default. Afterwards the save and print routines
223  * will export/print all graphs, not just the current one. There will be no additional border between the graphs, as the
224  * class expects the internal graph borders to be sufficient.
225  *
226  * \see \ref JKQTPLOTTER_SYNCMULTIPLOT
227  *
228  *
229  * \section jkqtplotter_base_defaultvalues Default Properties
230  * The plot is configured by a huge set of properties. For each property there is also a second protected variable which
231  * contains its default value. This way it is possible to store only those parameters in an INI file which have changed with
232  * respect to the default values. If the property is called \c property then the according default value is stored in
233  * \c default_property. To reduce the code to be entered you can use the JKQTPPROPERTY() macro.
234  *
235  * Default values are available only for properties that control the appearance of the graphs (line widths, border widths,
236  * color, fonts ...), not the type of the graph (xmin, xmax, logXAxis, ...)
237  *
238  * \section jkqtplotter_base_userprops User Properties
239  * There is a subset of options that describe how the user interacted with the plotter (export/print scaling factors etc, save directories,
240  * other export settings, ...). These are not stored/loaded using saveSettings() and loadSettings(), but using saveUserSettings() and loadUserSettings().
241  * These methods MAY (strictly optional and turned off by default) be called by saveSettings() and loadSettings(), if the property userSettigsFilename ( \copybrief userSettigsFilename )is
242  * set (not-empty). In this case the suer settings are stored/loaded also everytime they are changed by the user or programmatically.
243  */
