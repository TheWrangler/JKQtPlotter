65  * This class is a QWidget-wrapper around JKQTBasePlotter. It uses the tools from JKQTBasePlotter
66  * to display scientific plots. This class mostly adds the Widget for the output and adds different
67  * types of user interactions.
68  *
69  * <b>Please have a look at the documentation of JKQTBasePlotter for details on the management of graphs
70  * and the formating/styling of the plot and graphs!</b>
71  *
72  * The rest of this documentation ins split into sections that each treat a special topic, as outlines below:
73  *
74  * \tableofcontents
75  *
76  *
77  * \section JKQTPLOTTER_SYNCMULTIPLOT Synchronizing Several Plots
78  *
79  * Often a single plot is not sufficient, but several plots need to be aligned with respect to each other:
80  *
81  * \image html test_multiplot.png
82  *
83  * This can be achieved by putting several JKQTPlotter instances into a
84  * <a href="http://doc.qt.io/qt-5/layout.html">Qt Layout</a>. Then you can fill each plot differently and
85  * set the x-/y-range of each plot by hand. This method works for simple cases, but has several drawbacks:
86  * - Due to the independent and automatic layouting of each plot, the axes do not need to be aligned properly
87  * - When you print the plot, the printing does not know about the layout and will only print one of the
88  * several plots in your layout.
89  * - when you zoom/pan in one of the plots (e.g. using the mouse), the other plots will not adapt their
90  * axes to match the new area, but especially in cases as in the image above it would be beneficial,
91  * that tha x-axis of the plot at the bottom follows the x-axis of the plot above etc.
92  * .
93  *
94  * To overcome these limitations, JKQTPlotter offers an API with which you can declare relations between
95  * different plots (one of them is made the master) and you can synchronize the axes of two plots, when
96  * zooming (also when calling e.g. zoomToFit() or setXY() ). This API is:
97  * - <b>Declaring the Relations (forwarding to JKQTBasePlotter !):</b>
98  * - synchronizeToMaster() / JKQTBasePlotter::synchronizeToMaster() synchronizes the parent JKQTPlotter with another JKQTPlotter. With two boolean-parameters
99  * you can specify the axes to be synchronized. E.g. in the case above, you would call:
100  * \code
101  * // synchronize width/x-axis of plotResid to width/x-axis of plotMain
102  * plotResid->synchronizeToMaster(plotMain, true, false, true, true);
103  *
104  * // synchronize y-axis of width/plotResidHist to y-axis of width/plotResid
105  * plotResidHist->synchronizeToMaster(plotResid, false, true, true, true);
106  * \endcode
107  * This will synchronize the x-axes of the top (\c plotMain ) and bottom-left plot (\c plotResid ),
108  * as well as the y-axes of the bottom-left (\c plotResid ) and bottom-right plot (\c plotResidHist ).
109  * After this call they will have the same size in screen pixels and always span the same range
110  * in plot coordinates.
111  * - resetMasterSynchronization() / JKQTBasePlotter::resetMasterSynchronization() deletes all synchronizations
112  * from the JKQTPlotter
113  * .
114  * - <b>Synchronizing Axes (forwarding to JKQTBasePlotter !):</b>
115  * - setGridPrinting() enables grid printing for this JKQTPlotter. If set to \c true , and you print afterwards,
116  * the printout (or export) will not only contain the plot itself, but also additional plots that were
117  * declared using addGridPrintingPlotter() (see below).
118  * - addGridPrintingPlotter() add a new plotter \a plotterOther for grid printing mode, at location \a x / \a y
119  * E.g. in the example shown above, you could call:
120  * \code
121  * plotMain->setGridPrinting(true);
122  * plotMain->addGridPrintingPlotter(0,1,plotResid);
123  * plotMain->addGridPrintingPlotter(1,1,plotResidHist);
124  * \endcode
125  * - clearGridPrintingPlotters() clear all additional plotters for grid printing mode
126  * .
127  * These two functionalities are kept separate, so you can use them independently.
128  *
129  * \note Note that the grid printing mode only allows to put plots to the right (positive x-values in addGridPrintingPlotter() )
130  * and to the bottom (positive y-values in addGridPrintingPlotter() ) of the current plot. Therefore the master plot
131  * needs to be the top-left plot of your grid and all plots need to be aligned in a grid (i.e. using
132  * <a href="http://doc.qt.io/qt-5/qgridlayout.html">QGridLayout</a>)
133  *
134  * \see See \ref JKQTPlotterMultiPlotLayout for an extensive example of the functionality.
135  *

185  * \subsection JKQTPLOTTER_ACTIONS QActions from a JKQTPlotter
186  * Often you want to allow the suer to operate a plot from a user-defined QToolBar or a QMenu/QMenuBar in your
187  * application (e.g. provide zooming commands ...). there are generally two ways to achieve this:
188  * <ol>
189  * <li> Simply connect home-brewn QAction instances to the slots provided by JKQTPlotter and JKQTBasePlotter.
190  * This also allows you to connect different plot properties to edit widgets in your forms.
191  * \code
192  * // add a checkbox to show and hide the position display label in the plot
193  * chkPositionDisplay=new QCheckBox(tr("show mouse cursor position"), this);
194  * chkPositionDisplay->setChecked(plot->isMousePositionShown());
195  * connect(chkPositionDisplay, SIGNAL(toggled(bool)), plot, SLOT(setMousePositionShown(bool)));
196  * \endcode
197  * </li>
198  * <li> For several functions (especially those also present in JKQTPlotter's context-emun, you can also find
199  * readily available QAction instances. these are available from JKQTBasePlotter (e.g. by JKQTBasePlotter::getActionPrint() ...).
200  * From JKQTPlotter you therefor have to use: <code>getPlotter()->getActionPrint()</code>
201  * \code
202  * // add some of the default QActions from the JKQTPlotter to the window menu
203  * // Some of the are also available in the context menu and toolbar of the JKQTPlotter
204  * // others are not
205  * QMenu* menuPlot=menuBar()->addMenu("Plot-Menu");
206  * menuPlot->addAction(plot->getPlotter()->getActionPrint());
207  * QMenu* menuPlotS=menuPlot->addMenu("Save ...");
208  * menuPlotS->addAction(plot->getPlotter()->getActionSaveData());
209  * menuPlotS->addAction(plot->getPlotter()->getActionSavePDF()); // not available from JKQTPlotter by default
210  * menuPlotS->addAction(plot->getPlotter()->getActionSavePlot());
211  * \endcode
212  * </li>
213  * </ol>
214  * \see \ref JKQTPlotterUserInteraction

218  * \subsection JKQTPLOTTER_USERMOUSEINTERACTION Mouse-Interaction in JKQTPlotter
219  *
220  * This section summarizes all user-interaction functions in JKQTPlotter that somehow relate to the mouse.
221  * These are:
222  * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEDRAG
223  * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSECLICK
224  * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEWHEEL
225  * - \ref JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEMOVE
226  * .
227  *
228  * \note Zooming and Paning interactions apply to both axes when they are performed with the mouse
229  * inside the plot. They are limited to one of the two axes, when the mouse hovers over that axis
230  * (e.g. when you zoom by mouse-wheel while the mouse pointer is below the plot, over the x-axis,
231  * only the x-axis is affected by the operation).
232  *
233  * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEDRAG Actions When Dragging the Mouse
234  * JKQTPlotter offers several methods that allow to customize, how it reacts to mouse actions:
235  * - registerMouseDragAction() tells JKQTPlotter to perform a certain action (selected from JKQTPlotter::MouseActionMode)
236  * when a specified mouse button is pushed while a specified (or no) keyborad modifiers (e.g. CTRL,ALT,SHIFT...) is pressed.
237  * By default JKQTPlotter calls these two registers in its constructors:
238  * \code
239  * registerMouseDragAction(Qt::LeftButton, Qt::NoModifier, JKQTPlotter::MouseActionMode::ZoomRectangle);
240  * registerMouseDragAction(Qt::LeftButton, Qt::ControlModifier, JKQTPlotter::MouseActionMode::PanPlotOnMove);
241  * \endcode
242  * Therefore by default you can draw a zoom rectangle with the left mouse button without modifiers
243  * and you can pan/drag the plot with the left mouse-button while keeping CTRL pressed.
244  * - deregisterMouseDragAction() deletes a previously defined user-interaction
245  * - clearAllRegisteredMouseDragActions() deletes all previously specified user-actions
246  * .
247  *
248  * Pressing the \c ESC key will stop the current JKQTPlotter::MouseActionMode.
249  *
250  * If e.g. the mode JKQTPlotter::MouseActionMode::ZoomRectangle is selected, while you drag the mouse, the
251  * zoom rectangle is drawn over the plot. You can modify the style of drawing using these functions:
252  * - setUserActionColor() sets the color of the drawn shape
253  * - setUserActionCompositionMode() specifies how to combine the shape with the existing plot
254  * .
255  *
256  * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSECLICK Actions After (Double-)Clicks on the Mouse Buttons
257  * The right mouse button has a special role: If it is single-clicked and no JKQTPlotter::MouseActionMode is specified
258  * for the vent, it opens the context menu, unless you call \c setContextMenuMoode(JKQTPlotter::NoContextMenu) .
259  * You can also use setContextMenuMoode() to specify which type of context menu is shown. See JKQTPlotter::ContextMenuModes
260  * for details on the available modes.
261  *
262  * For any mouse-click, one of the following signals is emitted:
263  * - plotMouseClicked() for any single-click (during the pressDown-Event!)
264  * - plotMouseDoubleClicked() for any double-click
265  * .
266  *
267  * The reaction to double-clicks of the mouse buttons can be specified separately. The possible
268  * actions are listed in JKQTPlotter::MouseDoubleClickActions. You can bind one of these actions
269  * to a specific click with these functions:
270  * - registerMouseDoubleClickAction() tells JKQTPlotter to perform a certain action (selected from JKQTPlotter::MouseDoubleClickActions)
271  * when a specified mouse button is pushed while a specified (or no) keyborad modifiers (e.g. CTRL,ALT,SHIFT...) is pressed.
272  * By default JKQTPlotter calls this in its constructors:
273  * \code
274  * registerMouseDoubleClickAction(Qt::LeftButton, Qt::NoModifier, JKQTPlotter::MouseDoubleClickActions::ClickMovesViewport);
275  * \endcode
276  * Therefore by default you can pan the plot/move the viewport by double clicking a location
277  * - deregisterMouseDoubleClickAction() deletes a previously defined user-interaction
278  * - clearAllRegisteredMouseDoubleClickAction() deletes all previously specified user-actions
279  * .
280  * The button to react to is specified as a parameter.
281  *
282  * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEWHEEL Actions When a Mouse Wheel Event Occurs
283  * The actions to be performed when the mouse hweel is operated are specified in JKQTPlotter::MouseWheelActions.
284  * You can bind one of these actions to the mouse-wheel (under the condition that a specified Qt::KeyboardModifier
285  * is pressed) by using these functions:
286  * - registerMouseWheelAction() tells JKQTPlotter to perform a certain action (selected from JKQTPlotter::MouseWheelActions)
287  * when a specified mouse button is pushed while a specified (or no) keyborad modifiers (e.g. CTRL,ALT,SHIFT...) is pressed.
288  * By default JKQTPlotter calls this in its constructors:
289  * \code
290  * registerMouseWheelAction(Qt::NoModifier, JKQTPlotter::MouseWheelActions::ZoomByWheel);
291  * \endcode
292  * Therefore by default you can zoom into and out of the plot, using the mouse wheel.
293  * - deregisterMouseWheelAction() deletes a previously defined user-interaction
294  * - clearAllMouseWheelActions() deletes all previously specified user-actions
295  * .
296  * In addition the signal void plotMouseWheelOperated() is emitted whenever a mouse-wheel event occurs.
297  *
298  *
299  * \subsubsection JKQTPLOTTER_USERMOUSEINTERACTION_MOUSEMOVE Signaling When Mouse Moves
300  * In addition the signal plotMouseMove() is called whenever the mouse moves over the plot.
301  * Additional signals may be emitted, depending on the currently active JKQTPlotter::MouseActionMode.
302  *
303  * Also the current mouse position is shown above the graph by default (can be switched on or off
304  * using setMousePositionShown() ).
305  *
